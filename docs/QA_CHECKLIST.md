# QA Checklist

每次改动主流程后，至少跑一遍这份手工验收。目标是防止产品重新滑回“工具页 / 后台系统”，以及防止路由和隐私规则回退。

## 首页 Feed

- [ ] 首页只展示图文动态流，不出现搜索框、分类 Tab、大功能入口、Issue、Board、Wiki 或 Group 入口
- [ ] 点击首页动态图片、正文或卡片主体，进入动态详情
- [ ] 首页动态里的头像和名字不能直接进入毛孩主页
- [ ] 首页只展示和我有关的毛孩动态，不出现陌生推荐、热门宠物或订阅流

## 动态详情

- [ ] 动态详情只展示当前这一条动态，不展示整只毛孩主页
- [ ] 动态详情可以通过“查看全部动态”进入对应毛孩主页
- [ ] 从首页进入动态详情后点击返回，应回到首页
- [ ] 从动态详情进入毛孩主页后点击返回，应回到刚才那条动态详情
- [ ] 多动物动态展示“这条动态里的毛孩”
- [ ] 点击“这条动态里的毛孩”中的任意毛孩头像，可以进入对应毛孩主页

## 毛孩主页

- [ ] 毛孩主页顶部没有大封面图
- [ ] 毛孩主页不显示关注、订阅、已关注或已订阅
- [ ] 毛孩主页默认进入“动态”Tab
- [ ] 毛孩主页保留轻量 Tab：动态、照片、地点、档案
- [ ] 图鉴进入毛孩主页后点击返回，应回到图鉴
- [ ] 地图进入毛孩主页后点击返回，应回到地图
- [ ] 只读毛孩不显示右下角发动态按钮
- [ ] 有记录权限的毛孩显示右下角小号发动态按钮
- [ ] 照片 Tab 不提供额外添加入口，新增内容统一走底部 `+` 或主页右下角 `+`

## 图鉴

- [ ] 图鉴是所有我能看到的毛孩集合
- [ ] 图鉴支持搜索和标签筛选
- [ ] 图鉴动物卡点击进入毛孩主页
- [ ] 图鉴中能看到“我的 / 分享给我的 / 允许我记录 / 已送养 / 朋友在记录”等轻量关系标签

## 发动态

- [ ] 底部 `+` 只出现三个一级动作：发动态、新建毛孩、导入照片
- [ ] 发动态可以选择一只或多只毛孩
- [ ] 发动态必须能设置主角毛孩
- [ ] 选择多只毛孩后，动态会出现在每只关联毛孩主页里
- [ ] 如果关联动物里包含流浪动物，位置隐私按流浪动物规则处理
- [ ] 切换主角后，记录类型和状态字段不会保留不适用的旧值

## 流浪动物隐私

- [ ] 流浪动物动态不展示精确坐标
- [ ] 流浪动物地点显示为模糊地点或文字描述
- [ ] 多动物动态只要包含流浪动物，就按流浪动物隐私规则处理
- [ ] 分享流浪动物动态或主页时，不出现精确位置、固定出没时间、幼崽窝点或抓捕计划
- [ ] shared_record 权限不能修改敏感位置

## 送养与关系

- [ ] 已送养毛孩仍可出现在原主人的图鉴中
- [ ] 朋友更新被送养毛孩后，原主人首页可以看到动态
- [ ] 动物档案 Tab 能展示亲子、同窝、朋友、常一起出现等关系
- [ ] 多动物动态能表达一家三口、同窝、一起遛弯等场景

## 我的页

- [ ] 我的页不出现 Issue、Board、Wiki、Group Management、CreateGroup、JoinGroup 等重型工作流入口
- [ ] 我的页使用“共享与权限”表达分享关系
- [ ] 协作圈只作为分享范围概念，不进入独立小组空间

## 构建与部署

- [ ] `npm run build` 通过
- [ ] GitHub Pages / 预览部署成功
- [ ] 手机浏览首页、动态详情、毛孩主页、图鉴、发动态时没有明显挤压、遮挡或误触
