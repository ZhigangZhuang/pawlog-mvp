# 产品需求文档：流浪猫狗档案 App

## 产品定位

毛孩档案是一个以流浪猫狗为核心的图文档案和安全协作记录 App。

主线：

- 给每只流浪猫狗建立长期主页
- 记录出现、喂养、健康观察、TNR、救助、可领养和送养过程
- 保护流浪动物敏感位置和固定出没信息
- 把某只毛孩主页分享给可信的人查看或一起记录
- 维护多只毛孩之间的关系，例如母子、同窝、朋友、常一起出现
- 在送养或转交后继续看到流浪动物后续成长

辅助场景：

- 顺手记录自家宠物的照片、日常、关系和送养/转交
- 支持“我家猫生小猫，送给朋友后继续查看成长”

本产品不是普通宠物记录 App，不是宠物版小红书，也不是公开宠物订阅平台。

核心差异化不是记录“我家宠物今天多可爱”，而是长期、安全、可协作地记录流浪猫狗的真实状态和关系网。

一句话定位：

> 一个专门记录流浪猫狗的图文档案 App，可以安全记录、持续追踪，也可以分享给可信的人一起更新。

## 核心对象

```text
Animal = 毛孩主页
Post = 图文动态
PostAnimal = 动态和多只毛孩的关联
Share / Circle = 分享范围和协作权限
```

核心原则：

```text
Feed 是流浪动物优先的动态流
Animal 是长期主页
Share 是可信分享范围
```

## 用户关系

MVP 不做关注、订阅、粉丝和陌生推荐。

动物和用户之间只保留三种主关系：

```ts
type AnimalRelation =
  | "owned"
  | "shared_view"
  | "shared_record";
```

页面文案统一使用：

- 我的
- 分享给我的
- 允许我记录

扩展关系用于送养和转交：

```ts
type AnimalOwnershipStatus =
  | "owned_by_me"
  | "shared_to_me"
  | "transferred_out"
  | "transferred_to_me"
  | "co_recording";
```

## 当前 MVP 页面

底部导航：

```text
首页 ｜ 图鉴 ｜ + ｜ 地图 ｜ 我的
```

页面职责：

- 首页：优先展示流浪猫狗动态，自家宠物动态作为补充
- 图鉴：找流浪动物、搜索、分类、筛选
- +：发动态、新建毛孩
- 地图：我的记录地点，不做公开地图
- 我的：数据、共享权限、工具和设置

## 核心功能

### 首页动态流

- 优先展示我记录的流浪猫狗动态
- 优先展示分享给我的流浪猫狗动态
- 优先展示允许我记录的流浪猫狗动态
- 优先展示需要观察、需要救助、待绝育、已绝育、可领养的流浪动物动态
- 自家宠物动态作为补充内容
- 不展示陌生推荐、热门宠物、全站发现、附近公开动物
- 不承担搜索和分类任务

### 动态详情

- 只展示当前这一条动态
- 展示图片、正文、类型、位置、来源
- 多动物动态展示“这条动态里的毛孩”
- 流浪动物动态分享时隐藏精确位置和敏感信息

### 毛孩主页

- 展示某只毛孩的长期记录
- 默认进入动态 Tab
- Tab 包括：动态、照片、地点、档案
- 不显示关注或订阅按钮
- 有记录权限时，右下角显示轻量发布入口
- 流浪动物主页优先突出最近出现、模糊地点、健康、亲人程度、绝育/TNR、救助、可领养、关联动物和隐私提示
- 自家宠物主页保留日常照片和关系记录，但不是 MVP 的主卖点

### 图鉴

- 展示所有我能看到的毛孩
- 支持搜索、标签筛选、网格/列表视图
- 默认筛选和统计偏向流浪猫狗
- 包含流浪猫、流浪狗、待观察、待绝育、已绝育、需救助、可领养、分享给我的、允许我记录、自家宠物等轻量标签

### 多动物动态

一条动态可以关联一只或多只毛孩。

示例：

- 猫妈妈、小白崽、小黑崽一起出现在公园
- 奶盖和三只小猫的家庭记录
- 豆包和 Lucky 一起散步

### 动物关系

支持轻量关系：

- parent
- child
- same_litter
- friend
- often_seen_together
- same_feeding_area

关系展示在毛孩主页的“档案”Tab 中。

### 送养 / 转交

送养和转交不是删除档案，而是改变记录关系。

场景：

- 小猫送给朋友
- 原主人保留查看权限
- 朋友继续记录成长
- 如果允许，原主人也可以继续补充旧照片或动态

## 非目标

MVP 不做：

- 以自家宠物成长相册为主线
- 复杂疫苗提醒、驱虫管理、体重趋势或宠物健康报告
- 宠物版小红书
- 公开宠物订阅
- 粉丝系统
- 热门榜
- 陌生人推荐
- 公开附近流浪动物地图
- 复杂小组空间
- Issue / Board / Wiki 主流程
- AI 识别
- 真实地图和云同步
