# 产品需求文档：毛孩档案 App

## 产品定位

毛孩档案是一个给每只毛孩建立长期主页的图文记录 App。

它可以用于：

- 私密记录自家宠物的成长、照片、健康、体重和纪念日
- 私密记录附近流浪猫狗的出现、喂养、健康观察、TNR 和救助进度
- 把某只毛孩主页分享给可信的人查看或一起记录
- 维护多只毛孩之间的关系，例如母子、同窝、朋友、常一起出现
- 在送养或转交后继续看到毛孩成长

本产品不是宠物版小红书，也不是公开宠物订阅平台。

核心差异化不是发现陌生宠物内容，而是长期、结构化、可协作地记录真实关系里的毛孩。

## 核心对象

```text
Animal = 毛孩主页
Post = 图文动态
PostAnimal = 动态和多只毛孩的关联
Share / Circle = 分享范围和协作权限
```

核心原则：

```text
Feed 是动态流
Animal 是长期主页
Share / Circle 是分享范围
```

## 用户关系

MVP 不做关注、订阅、粉丝和陌生推荐。

动物和用户之间只保留三种主关系：

```ts
type AnimalRelation =
  | "owned"
  | "shared_view"
  | "shared_record";
```

页面文案统一使用：

- 我的
- 分享给我的
- 允许我记录

扩展关系用于送养和转交：

```ts
type AnimalOwnershipStatus =
  | "owned_by_me"
  | "shared_to_me"
  | "transferred_out"
  | "transferred_to_me"
  | "co_recording";
```

## 当前 MVP 页面

底部导航：

```text
首页 ｜ 图鉴 ｜ + ｜ 地图 ｜ 我的
```

页面职责：

- 首页：只展示和我有关的毛孩动态
- 图鉴：找毛孩、搜索、分类、筛选
- +：发动态、新建毛孩、导入照片
- 地图：我的记录地点，不做公开地图
- 我的：数据、共享权限、工具和设置

## 核心功能

### 首页动态流

- 展示我的毛孩动态
- 展示分享给我的毛孩动态
- 展示允许我记录的毛孩动态
- 不展示陌生推荐、热门宠物、全站发现、附近公开动物
- 不承担搜索和分类任务

### 动态详情

- 只展示当前这一条动态
- 展示图片、正文、类型、位置、来源
- 多动物动态展示“这条动态里的毛孩”
- 流浪动物动态分享时隐藏精确位置和敏感信息

### 毛孩主页

- 展示某只毛孩的长期记录
- 默认进入动态 Tab
- Tab 包括：动态、照片、地点、档案
- 不显示关注或订阅按钮
- 有记录权限时，右下角显示轻量发布入口

### 图鉴

- 展示所有我能看到的毛孩
- 支持搜索、标签筛选、网格/列表视图
- 包含我的、分享给我的、允许我记录、已送养、家庭/同窝等轻量标签

### 多动物动态

一条动态可以关联一只或多只毛孩。

示例：

- 猫妈妈、小白崽、小黑崽一起出现在公园
- 奶盖和三只小猫的家庭记录
- 豆包和 Lucky 一起散步

### 动物关系

支持轻量关系：

- parent
- child
- same_litter
- friend
- often_seen_together
- same_feeding_area

关系展示在毛孩主页的“档案”Tab 中。

### 送养 / 转交

送养和转交不是删除档案，而是改变记录关系。

场景：

- 小猫送给朋友
- 原主人保留查看权限
- 朋友继续记录成长
- 如果允许，原主人也可以继续补充旧照片或动态

## 非目标

MVP 不做：

- 宠物版小红书
- 公开宠物订阅
- 粉丝系统
- 热门榜
- 陌生人推荐
- 公开附近流浪动物地图
- 复杂小组空间
- Issue / Board / Wiki 主流程
- AI 识别
- 真实地图和云同步
