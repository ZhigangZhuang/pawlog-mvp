# UX 规则

本文件记录不可轻易违背的交互规则。后续改 UI 前，应先检查这里。

## 信息架构

```text
首页 = 看动态
图鉴 = 找毛孩 / 搜索 / 分类 / 筛选
地图 = 按地点找记录
我的 = 数据、共享和设置
```

首页不承担搜索、分类、筛选、任务看板或工作流入口。

## 首页 Feed

- 首页只展示图文动态流
- 首页不做搜索框
- 首页不做分类 Tab
- 首页不放大功能按钮
- 首页不展示 Issue、Board、Wiki、小组空间入口
- 首页只展示和我有关的毛孩动态
- 首页不做陌生推荐、热门宠物或公开订阅流

点击规则：

```text
首页 Feed 卡片主体 -> 动态详情
首页 Feed 头像/名字 -> 动态详情
首页 Feed 图片/正文 -> 动态详情
```

当前规则是：**首页不直接进入毛孩主页，头像也不例外**。

毛孩主页入口只来自：

- 图鉴动物卡
- 动态详情里的“查看毛孩主页”
- 地图 Bottom Sheet 的“查看主页”

## 动态详情

- 动态详情只展示当前这一条动态
- 返回应回到来源页
- 从动态详情进入毛孩主页后，毛孩主页返回应回到这条动态
- 多动物动态必须展示“这条动态里的毛孩”
- 点击动态详情中的关联毛孩头像，可以进入对应毛孩主页

## 毛孩主页

- 毛孩主页不是社交账号主页，也不是后台表单详情页
- 顶部不使用大封面图
- 顶部只保留圆形主图、昵称、轻量状态和统计
- 不显示关注按钮
- 不显示订阅按钮
- 默认进入“动态”Tab
- 档案字段放到“档案”Tab，不在首屏铺满
- 照片 Tab 不再放新增照片按钮

Tab：

```text
动态 ｜ 照片 ｜ 地点 ｜ 档案
```

## 添加入口

底部导航只保留一个主新增入口 `+`。

一级动作只保留：

```text
发动态
新建毛孩
导入照片
```

不要在首页、图鉴页、毛孩主页到处铺大按钮。

## 发动态流程

发动态应像发一条图文记录，而不是填复杂表单。

流程：

```text
点击 +
选择发动态
选择一只或多只毛孩
设置主角毛孩
添加图片
写内容
可选：类型 / 地点 / 标签
发布
```

如果选择了流浪动物，应提示位置隐私。

## 图鉴

- 图鉴负责搜索、分类、筛选
- 默认网格视图
- 可以切换列表视图
- 顶部允许搜索框和轻量标签栏
- 不做大统计面板
- 不做任务工作台

图鉴卡片点击：

```text
图鉴动物卡 -> 毛孩主页
```

## 分享和权限

不使用以下概念：

- 关注
- 已关注
- 订阅
- 已订阅
- 我订阅的毛孩
- 粉丝

统一使用：

- 我的
- 分享给我的
- 允许我记录

分享动物主页时，MVP 只提供：

- 只读查看
- 允许添加动态

## UI 风格

- 图片优先
- 文字轻量
- 避免大按钮堆叠
- 使用小图标、小胶囊、小菜单
- 标签以 `#标签` 轻量展示
- 不做后台系统感
- 不做项目管理感
- 不做公开社区感
