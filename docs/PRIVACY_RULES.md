# 隐私与安全规则

本产品涉及流浪动物位置、救助进度、送养关系和个人分享权限，必须默认保护隐私。

## 基本原则

- 所有档案默认私密
- 分享前默认过滤敏感信息
- 流浪动物位置默认模糊
- 协作权限默认克制
- 不做公开附近流浪动物地图

## 流浪动物安全红线

禁止：

- 公开精确坐标
- 公开幼崽窝点
- 公开固定出没时间
- 公开抓捕计划
- 公开临时安置地点
- 公开投喂人身份
- 做公开附近流浪动物推荐
- 做陌生人可浏览的精确地图

## 位置规则

流浪动物位置默认：

```text
precision_level = "text_only" 或 "blurred"
visibility = "private"
```

敏感地点必须默认私密：

- 幼崽窝点
- 未绝育母猫常出现点
- 受伤动物位置
- 固定睡觉地点
- 固定喂食点
- 抓捕计划地点
- 临时安置地点

公开分享时必须移除：

- latitude
- longitude
- 精确地址
- 楼栋门牌
- 固定时间
- 敏感备注

## 分享权限

分享动物主页时，MVP 只允许：

- 只读查看
- 允许添加动态

`shared_record` / `can_post` 权限可以：

- 添加照片
- 添加文字动态
- 添加喂养记录
- 添加健康观察
- 添加模糊出现记录

不能：

- 删除原档案
- 修改所有权
- 修改敏感位置
- 公开精确位置
- 删除别人记录
- 修改核心档案字段

## 多动物动态

多动物动态中，只要包含流浪动物，就按流浪动物隐私规则处理。

例如：

```text
豆包 + Lucky：普通宠物动态，可展示普通地点
猫妈妈 + 小白崽：流浪动物动态，地点必须模糊
```

如果动态中包含幼崽、受伤动物或未绝育母猫，位置默认敏感。

## 动物关系

动物关系可以展示，但必须克制。

允许展示：

- 母子
- 同窝
- 朋友
- 常一起出现
- 同一喂养点

避免展示：

- 会暴露幼崽窝点的细节
- 会暴露固定路线的细节
- 会暴露救助人、领养人真实身份的内容

## 送养 / 转交

送养和转交记录中，个人信息要克制展示。

推荐展示：

- 朋友 A
- 邻居
- 家人
- 临时照顾人

不推荐默认展示：

- 真实姓名
- 手机号
- 住址
- 社交账号
- 详细家庭地址

## 地图

地图页只展示：

- 我的记录地点
- 我有权限看到的模糊地点

不做：

- 公开地图
- 附近流浪动物推荐
- 陌生人查看精确点位
