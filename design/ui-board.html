产品原则

先锁规则，再继续做功能。

首页只看动态首页不做搜索、分类、任务入口。任何 Feed 点击都进入动态详情。
Animal 是主页毛孩主页展示长期动态、照片、地点和档案，不做关注/订阅。
分享是权限统一使用：我的、分享给我的、允许我记录。
流浪动物安全优先位置默认模糊，不公开幼崽窝点、固定出没时间或抓捕计划。

Design Tokens

温暖、干净、轻手账感，不做后台系统。

全功能流程演示

6 个核心流程，说清楚产品怎么用、和普通宠物相册有什么不同。

🐾

这不是宠物版小红书，也不是自家宠物成长相册。
核心场景是：长期、安全、可协作地记录流浪猫狗的真实状态和关系网。自家宠物只作为辅助场景。
产品差异：没有公开推荐、没有粉丝系统、位置默认模糊、分享给可信的人而非陌生人。

1

主流程：记录流浪猫

打开 App 到发出第一条动态，最快 3 步。

  1. 1打开首页，看到时间流动态（左侧时间点 + 右侧缩略图，首屏 3-4 条）
  2. 2点击一条动态，进入动态详情页
  3. 3点击动物头像 / 名字，进入毛孩主页
  4. 4在主页查看动态、照片、地点、档案四个 Tab
  5. 5点击右下角 +，给这只毛孩发一条新动态
首页时间流 动态详情 毛孩主页 发动态
首页只展示我记录的和分享给我的，不做公开发现、热门推荐或陌生动物推送。
2

新建流浪动物

给第一次见到的流浪猫狗建立档案。

  1. 1点击底部 +，选择「新建毛孩」
  2. 2选择流浪猫 / 流浪狗（自动跳过品种、生日等非必填字段）
  3. 3填写昵称、毛色、外貌特征（便于下次识别）
  4. 4设置健康状态、绝育状态、是否亲人
  5. 5保存后直接进入这只毛孩的主页，开始记录
新建毛孩 流浪动物字段 绝育状态
流浪动物不要求品种、生日、回家日期。字段尽量少，降低建档门槛，让记录习惯先养成。
3

多动物动态

一条动态关联多只毛孩，自动同步到各自主页。

  1. 1点击底部 +，选择「发动态」
  2. 2上传照片，写一句记录
  3. 3毛孩选择器默认显示最近记录的 6 只，勾选猫妈妈、小白崽、小黑崽
  4. 4点击已选 chip 设置主角：猫妈妈
  5. 5发布后，这条动态同时出现在三只毛孩的主页动态 Tab
  6. 6动态详情显示「这条动态里的毛孩」区（单动物时隐藏）
多选毛孩 设置主角 动态同步
主角决定动态的封面和顶部头像。选择器只展示最近 / 常记录的毛孩，支持搜索和「查看全部」。
4

送养 / 转交后继续记录

档案不删除，关系变了，记录还在。

  1. 1在小猫主页点击「···」，选择「送养 / 转交」
  2. 2填写对方称呼（如「朋友 A」），选择转交类型（送养 / 寄养 / 转交）
  3. 3权限设置：是否保留查看 / 是否允许我继续补充动态
  4. 4转交后对方开始记录小猫成长
  5. 5原主人首页仍然能看到小猫的新动态（分享给我的）
送养流程 保留查看权 转交后记录
送养不是删除档案，而是改变记录关系。原主人可以保留「只读查看」或「继续补充旧照片」的权限。
5

分享给可信的人

把一只流浪猫的档案共享给邻居或志愿者一起记录。

  1. 1在毛孩主页点击「↗」分享
  2. 2选择权限：只读查看 / 允许添加动态
  3. 3生成分享链接，发给邻居 / 志愿者
  4. 4对方打开后选择「保存到我的图鉴」
  5. 5如果允许添加，对方可以继续补充喂养、健康、出现记录
分享链接 只读 / 可记录 保存到图鉴
分享对象是手动选择的可信联系人，不是公开订阅。对方不会自动出现在「推荐」或「发现」列表里。
6

流浪动物隐私保护

位置默认模糊，敏感信息分享前自动过滤。

  1. 1创建流浪动物地点记录时，位置只填文字描述（如「北门附近」），不存储精确坐标
  2. 2发动态时位置字段自动标注「模糊地点」
  3. 3分享动态时自动过滤精确位置、固定出没时间和敏感备注
  4. 4公开分享卡只展示动物外貌和健康摘要，不含地图坐标
  5. 5地图页仅展示我的私密记录，不做公开附近流浪动物地图
位置模糊 分享过滤 私密地图
这是产品的安全底线。精确地点、幼崽窝点、抓捕计划均不可公开，也不建议写入动态正文。
